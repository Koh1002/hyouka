# hyouka — LLM出力の人手スコアリング（汎用・スマホ対応）

LLMを動かした後の出力を、**人手で少しずつ・迷わず・スマホからも**採点するための軽量Webアプリ。
バックエンド不要（静的サイト）。回答は端末の localStorage に自動保存され、いつでも **CSV（Excelで開ける）/ TSV（Excelに貼れる）** に書き出せる。

## 使い方（採点する人）
1. 公開URL（GitHub Pages）をスマホ/PCで開く。
2. 1件ずつ表示される内容を読み、スコアのボタンを押す（押すと自動で次へ）。必要ならメモ。
3. 中断OK。再度開くと続きから。`未回答へ` で次の未回答にジャンプ。
4. 終わったら（または途中でも）`CSV書き出し` で保存、または `TSVコピー` してExcelに貼り付け。

> 公開方法: リポジトリ Settings → Pages → Deploy from a branch → `main` / `/(root)` を選ぶと
> `https://koh1002.github.io/hyouka/` で開ける。

## 別タスクで使い回す（最重要：汎用設計）
**コードは触らず、データ2ファイルを差し替えるだけ。**
1. `data/あなたのタスク.json` … 採点対象の配列（各要素は任意のフィールドを持つオブジェクト）。例:
   ```json
   [ { "id": "ex-001", "prompt": "...", "model_output": "...", "category": "A" }, ... ]
   ```
2. `config.json` … 表示と採点軸の定義（コード非依存）:
   - `items_file` … データのパス（例 `data/あなたのタスク.json`）
   - `id_field` … 一意IDのフィールド名
   - `meta_fields` … CSVに一緒に出したい付随フィールド（集計・突合用）
   - `display` … 画面に出す項目。`type` は `title` / `text` / `longtext` / `badge`
   - `question` … 採点の問い
   - `options` … スコア選択肢 `{value,label,color}`（2値でも5段階でも可）
   - `allow_note` / `note_label` … 自由メモ欄

これで「要約の良し悪し」「有害性ラベル」「回答の正誤」など、**任意のLLM評価タスク**に転用できる。

## ファイル
- `index.html` … アプリ本体（設定駆動・依存なし）
- `config.json` … 現在のタスク設定（C1=回収判定の人手検証）
- `data/c1_items.json` … C1の採点対象（agentレポート×洞察ペア）
- 集計（人手 vs LLM判定器の一致）は、書き出したCSVと別途の判定器スコアを突合して算出。

## 補足
- 完全クライアント側。回答は端末ローカル保存→CSV/TSVで取り出す方式（トークン不要・安全）。
  リポジトリへ自動コミットしたい場合のみ別途バックエンド/Actionが必要（任意）。
